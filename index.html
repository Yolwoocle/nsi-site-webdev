Comment fonctionne un site web?

Tout le monde connait le Web. Mais saviez vous comment, dans les coulisses, fonctionnent les sites ?

Le HTML: Le squelette du site

Tout d'abord, le langage qu'utilisent les sites afin d'afficher leur contenu est le HTML. C'est un langage de balisage permettant de structurer les pages, d'inclure du contenu média, tel que des vidéos et des images, et d'insérer ce qu'on appelle de l'hypertexte.

Voici un exemple: 

<!DOCTPYE html>
<html>
    <head>
        <title>Les cactus c'est trop mignon!</title> 
    </head>
    <body>
        <h1>Bienvenue sur le site des fans de cactus !</h1>
        <a href="https://fr.wikipedia.org/wiki/Cactaceae">
            Article Wikipédia sur les cactus
        </a>
        <p>
            Les cactus sont <i>trop bien</i>, on peut leur faire des <b>câlins!</b>
        </p> 
        <img src="cactus.jpg">
    </body>
</html>

Ceci nous donnera:

Bienvenue sur le site des fans de cactus !

HTML fonctionne par un système de balises. Elles se présentent sous la forme <balise attribut="valeur">Contenu</balise>.

Dans cet exemple, <!DOCTYPE html> nous permet d'indiquer que la page web utilisera la dernière version de HTML.

Ensuite, tout le contenu de notre site sera contenu entre les balises <html>, obligatoirement.

Le <head> contient ce qu'on appelle les méta-données: les informations à propos du site, tel que son titre, son encodage, etc.

Le <body> est là où se trouve la majorité du contenu de la page: le texte, les liens, les images, etc. Par exemple:

Un <h1> indique un grand titre, et <h2> un titre un peu moins grand, <h3>, encore moins grand, ainsi de suite jusqu'à <h6>. 

<body>
    <h1>Titre très important!!!</h1>
    <h2>Titre important</h2>
    <h3>Titre un peu moins important</h3>
    <h4>Titre pas très important</h4>
    <h5>Titre peu important</h5>
    <h6>Titre inutile</h6>
</body>

La balise <a> indique un hyperlien: c'est tout simplement un lien permettant d'aller vers une autre page du site, ou des sites externes, comme ceci.

href="https://fr.wikipedia.org/wiki/Cactaceae" est un attribut: c'est une propriété de la balise. Ici, href indique le chemin qui amène vers le fichier du site ou d'un lien externe. Mais il existe des centaines d'autres attributs, tel que height, pour définir la hauteur d'une image ou d'une balise.

Une balise <img> indique l'emplacement d'une image. Elle est dite autofermante: on n'a pas besoin de la refermer avec une balise fermante (</img> n'existe pas).

Un coup de peinture: le CSS!

Le CSS, pour Cascading Style Sheets, est un langage décrivant la présentation d'une page Internet. Il nous permet par exemple de changer la police d'écriture, les couleurs d'arrière-plan, la taille de la police, etc.
La balise <style> nous permet d'insérer du CSS dans notre site: 

<style>
    body{
        background-color: greenyellow;
        font-family: Verdana;
    }
    img{
        width:100px
    }
</style>

Bienvenue sur le site des fans de cactus !

Dans cet exemple, on indique que on fera des modifications au body et à tous les éléments img. Ensuite, toutes les modifications qu'on souhaite apporter seront contenues entre les accolades {}. Ça indique quelles balises on souhaite modifier.

background-color, font-family, et width sont des propriétés, associées à une ou des valeurs. Elles indiquent ce que l'on souhaite modifier, respectivement: la couleur de l'arrière-plan, la police d'écriture, et la largeur de l'image. Il existe des centaines d'autres propriétés, certaines permettent de changer le formatage du texte, de rajouter des ombres, des contours, et même de faire des animations simples.

La relation entre le client et le serveur

Avant toute chose, nous devons définir ce qu'est le Web et Internet.
Le Web est différent d'Internet. Le Web utilise les services d'Internet, à la manière d'un taxi (le Web) qui utilise le réseau routier d'une ville (Internet). Le Web comprend des sites comme Google et Wikipédia. Internet, quant à lui, peut être par exemple utilisé par un servcie d'email.

Quand vous accédez à un site, votre appareil, le client, par exemple un natigateur web, envoie une requête vers un serveur, puis va attendre une réponse de sa part.

Le serveur, quant à lui, attend constamment qu'on le sollicite, et dès qu'il reçoit une requête, va générer une réponse, qu'il revoie au client.

Cette réponse peut inclure :

  • Les fichiers HTML et CSS d'un site
  • Les images, vidéos, fichier audio, etc.
  • Des scripts: des programmes permettant de créer des éléments interactifs et d'exécuter du code sur la page.

Les requêtes et les réponses sont transmises par un protocole: un ensemble de règles dictant comment sont transmises les informations. Sur le Web, le protocole usilisé est HTTP: HyperText Transfer Protocol. C'est ce qui apparait dans quasiment toutes les adresses de sites: par exemple, dans https://www.ecosia.com/ (Le "s" signifie "secure", la transmission est encryptée).
